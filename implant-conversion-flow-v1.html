Baseline Acknowledgment

保留原本的醫療信任語氣、青綠色系與「預約評估」框架； 但把一進頁就要求選分店與日期，改成先做低承諾 quick check，再收聯絡方式。

Keeping

  • 醫療感的 teal palette，維持專業與安心。
  • 主軸仍是預約評估，不改成折價促銷型 landing page。
  • 社會證明與醫師信任內容保留，但前移到 CTA 前。

Changing

  • 首屏文案改成 ad-message continuity，先承接「我想知道自己適不適合」。
  • 把日期選擇改為聯絡時段選擇，降低決策成本。
  • 表單從前置流程改成第 2 步，且縮到最小必要欄位。

Inferred Design System

  • Font: `Noto Sans TC` + `DM Sans` for numeric emphasis
  • Primary: `#0C8B8F`, Primary strong: `#096F73`
  • Accent: `#F2B44D`, Ink: `#14313B`, Bg: `#F4F6F2`
  • Spacing base: `8px`, Radius scale: `12 / 16 / 20 / 28`
  • Shadow style: soft medical-card elevation, not e-commerce hard shadows

State Matrix

  • Primary state 1: Landing default with quick qualification
  • Primary state 2: Contact sheet open with prefilled summary chips
  • Primary state 3: Success confirmation after submit
  • Secondary state: FAQ expanded/collapsed
  • Secondary state: different need/time selections

Why This Should Convert Better

  • 先讓使用者感到被理解，再要求輸入資料，符合高風險醫療決策心理。
  • 把「預約」拆成 micro-commit，先完成 2 題選擇，心理上更容易進到表單。
  • 強調專人協助安排，避免讓人誤以為現在就要做完整看診排程。
  • CTA 前就回應三個核心疑慮：適不適合、會不會被強推、接下來怎麼走。
